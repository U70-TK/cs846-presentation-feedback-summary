# CS846 Presentation — Guidelines Summary (Feedback + Consolidation)

This repo collects our original Week 10 code review / PR prompting guidelines, plus class feedback on what did/didn’t work, and consolidated “updated” versions proposed by other groups.

## Files

- **`src/guidelines/summarizedGuidelines.md`**: The place where **summarized guidelines should be added**.
  - This file intentionally includes space/placeholders for **all guidelines** from the original document to be filled in over time.
  
- **`src/guidelines/guidelines.md`**: The **original** guideline document created by our group during the presentation.

- **`src/guidelines/guidelineThatDidntWork.md`**: Guidelines that **did not work well** for classmates (with notes/examples).

- **`src/guidelines/newCounterGuidelines.md`**: A consolidated document that contains **all original guidelines** along with **updated versions proposed by each group**.
  - Several groups updated the *same* original guideline (e.g., regression testing). In those cases, all updated variants for that original guideline are grouped together.
  - The consolidated updates are placed **directly below** the corresponding original guideline.
  - The formatting/structure is explained at the top of the file for clarity.

- **`src/guidelines/workingGuidelines.md`**: Guidelines that **worked well**.
  - The top of the file contains the guidelines that worked.
  - At the bottom, under **“Add new summarized guidelines here”**, we add the **summarized** version(s) of guideline(s) (keeping the **same guideline number** as in `src/guidelines.md`).

- **`src/artifacts/`**: Contains code files.

- **`src/problem_descriptions/`**: Contains `problems.md` where we can add our problem and task descriptions.
